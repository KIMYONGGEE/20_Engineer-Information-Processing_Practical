## 2.연계 메커니즘 구성
- 연계 메커니즘
  - 응용 소프트웨어와 연계 대상 모듈 간의 데이터 연계 시 요구사항을 고려한 연계방법과 주기를 설계하기 위한 메커니즘
  - 연계 방식
    - 직접 연계
      - 단순 용이
      - 선응이 좋음
      - 결합도가 높음
      - 암복호화 처리 불가
      - 기술
        - DB 링크
        - DB 연결
        - API
        - JDBC
        - 하이퍼 링크
    - 간접 연계
      - 상이한 네트워크, 프로토콜 연계 가능
      - 인터페이스 변경 시 장애나 오류 없음
      - 매커니즘 복잡해 성능 저하 우려
      - 테스트 기간이 김
      - 기술
        - EAI
        - Web Service
        - ESB
        - Socket
- 연계 메커니즘 수행 절차 
  1. 연계데이터 추출 및 생성
  2. 코드 매핑 및 데이터 변환
  3. 연계 테이블/파일 생성
     - 송신 시스템 연계 테이블
       - 연계데이터 생성 일시
       - 송신 일시
       - 송신 상태
     - 수신 시스템 연계 테이블
       - 수신 일시
       - 처리 일시
       - 수신 상태
  4. 로그 기록
     - 송/수신 시스템에서 처리되는 일련의 과정에 대한 결과
  5. 연계 서버 또는 송/수신 어댑터
  6. 전송 단계
     - 전송 주기
       - 송신 시스템의 연계 데이터 추출 및 생성 방식
       - 연계 서버나 송수신 어댑터의 연계 주기
       - 수신 시스템의 연계 데이터를 운영 DB에 반영하는 방식 
  7. 운영 DB에 연계 데이터 반영
      - 매핑 프로그램을 별도 구현
#### 예상문제
1. 연계 메커니즘의 개념을 서술하시오.
    <details>
        <summary>답</summary>
        응용 소프트웨어와 연계 대상 모듈 간의 데이터 연계 시 요구사항을 고려한 연계방법과 주기를 설계하기 위한 흐름이다.
    </details>
2. DB 링크에 대해 서술하시오.
    <details>
        <summary>답</summary>
        수신 시스템에서 DB 링크를 생성하고 송신 시스템에서 해당 DB링크를 직접 참조한 방식이다.
    </details>
3. 연계 서버 장애 시 대응하는 주요 점검 사항을 서술하시오.
    <details>
        <summary>답</summary>
        연계 서버 엔진에서 기록된느 로그 파일의 내용확인 <br>
        연계 서버의 정상적 실행 여부 <br>
        연계 서버의 데이터 전송 변환 과정의 오류 여부 <br>
        송/수신 대상 시스템의 정상 연결 여부 등
    </details>
4. SOAP에 대해 서술하시오.
    <details>
        <summary>답</summary>
        HTTP, HTTPS, SMTP 등을 사용하여 XML 기반의 메시지를 네트워크 상태에서 교환하는 프로토콜이다.
    </details>

### Keyword
<details>
    <summary>인스턴스</summary>
    객체지향 프로그래밍에서 해당 클래스의 구조로 컴퓨터 저장 공간에서 할당된 실체
</details>
<details>
    <summary>SOAP</summary>
    HTTP, HTTPS, SMTP 등을 사용하여 XML 기반의 메시지를 네트워크 상태에서 교환하는 프로토콜
</details>
<details>
    <summary>이행함수 종속</summary>
    3차 정규화 중 A->B 이고, B->C 이면서 A->C 인 관계
</details>
<details>
    <summary>트리거</summary>
    특정 테이블에 삽입, 수정, 삭제등의 데이터 변경 이벤트가 발생시 DBMS에서 자동적으로 실행 되도록 구현된 프로그램
</details>
<details>
    <summary>오브젝트</summary>
    저장공간에서 할당되어 값을 가지거나 식별자에 의해 참조되는 공간
</details>
